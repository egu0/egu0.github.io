# CSS 重点总结

# CSS(3) 三大特性

# 层叠性

相同选择器设置相同的样式，此时其中一个会覆盖（层叠）另一个。

层叠性原则：

  • 样式冲突，遵循就近原则，那个样式离结构近就执行哪个样式
  • 样式不冲突，则不会层叠

示例

div {
  color: red;
  font-size: 18px;
}
div {
  /* 覆盖掉上边的 color */
  color: black; 
}

# 继承性

子标签会继承父标签的某些样式（主要是文字相关的样式，以 text-、font- 和 line-开头的属性），恰当使用可以简化代码、降低 CSS 样式的复杂性

速记：子承父业

示例

<style>
div {
  color: red;
  font-size: 18px;
}
</style>

<div>
  <!-- p 元素继承 div 元素样式 -->
  <p>猜猜我的文本颜色</p>
</div>

# 优先级

问题：给一个元素指定多个选择器，最终会使用哪个里边的样式呢

<style>
.box {
  color: blue;
}
div {
  color: red;
}
</style>

<div class=".box">
  猜猜我的文本颜色
</div>

上边两个选择器不是同类型选择器，无法使用层叠性解释，产生了优先级问题

多个选择器之间的优先级顺序

选择器 权重
继承 或 * 0
标签选择器、伪元素选择器 1
类选择器、伪类选择器、属性选择器 10
ID选择器 100
行内样式 style ='' 1000
!important 无穷大

回顾： 伪元素选择器(1)：E::before、E::after 属性选择器(10)：E[attr]、E[attr="val"]、E[attr^="val"]。。 伪类选择器(10)：E:first-child、E:nth-child(n)。。

当使用复合选择器时，会出现权重叠加。比如

div ul li       ->     3
.nav ul li      ->    12
a:hover         ->    11
.nav a          ->    11

重要（易忘️）：继承的权重为 0。 示例：判断 p 中文本颜色

<style>
#father {
  color: blue;
}
p {
  color: red;
}
</style>

<div id="father">
  <!-- 冲突：p 选择器定义的颜色样式和继承自父元素的样式 -->
  <!-- 结果是红色 -->
  <p>猜猜我的文本颜色</p>
</div>

另一个示例：如何 a 标签不能继承父元素的文本颜色

<style>
body {
  color: red;
}
</style>

<body>
  <!-- a 从 body 继承的 color 权重为 0，但仍然显示蓝色 -->
  <!-- 说明浏览器内部已经定义了 a { color: blue; } -->
  <a href="#">link</a>
</body>

# 总结

  • 层叠性：同种元素选择器包含冲突样式时，采用就近原则判断（离结构的距离）
  • 继承性：子元素会继承父元素的某些属性
  • 优先级：不同类型的选择器或复合选择器包含冲突样式时，需按照权重级别计算优先级
